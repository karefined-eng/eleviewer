# GitHub Setup Guide for EleViewer

This guide will walk you through creating a GitHub repository for your eleviewer project.

## Step 1: Create a GitHub Account (if you don't have one)

1. Go to https://github.com/signup
2. Create an account
3. Verify your email

Done! You now have a GitHub account.

---

## Step 2: Create a Repository on GitHub

1. Go to https://github.com/new (or click "New" → "Repository")
2. Fill in the form:
   - **Repository name**: `eleviewer`
   - **Description**: `A Windows text editor supporting DOCX, XLSX, MD, and TXT files`
   - **Visibility**: Public (if you want others to find it) or Private (if just for you)
   - **Initialize repository**: Leave unchecked (we'll do it locally)

3. Click **"Create repository"**

You now have an empty GitHub repository at:
```
https://github.com/YOUR_USERNAME/eleviewer
```

---

## Step 3: Push Your Local Code to GitHub

### On Your Laptop

1. **Open terminal/PowerShell** in your eleviewer project folder:
   ```bash
   cd C:\Users\kwadw\OneDrive\Documents\viewer_project
   ```

2. **Initialize Git locally** (one-time setup):
   ```bash
   git init
   git add .
   git commit -m "Initial commit: EleViewer - multi-format text editor"
   ```

3. **Add the remote repository** (connect to GitHub):
   ```bash
   git remote add origin https://github.com/YOUR_USERNAME/eleviewer.git
   ```
   
   ⚠️ **Replace YOUR_USERNAME with your GitHub username**

4. **Push to GitHub**:
   ```bash
   git branch -M main
   git push -u origin main
   ```

5. **Authenticate** (first time only):
   - If prompted, sign in with your GitHub credentials
   - Or use a Personal Access Token (GitHub will guide you)

Done! Your code is now on GitHub.

## Step 4: Files to Include

Make sure these files are in your project **before** pushing:

```
eleviewer/
├── main.py                      ✅ Include
├── ui.py                        ✅ Include
├── editor.py                    ✅ Include
├── file_handler.py              ✅ Include
├── docx_viewer.py               ✅ Include
├── xlsx_viewer.py               ✅ Include
├── session_manager.py           ✅ Include
├── quick_switcher.py            ✅ Include
├── pinned_files.py              ✅ Include
├── recent_files.py              ✅ Include
├── autosave.py                  ✅ Include
├── settings.py                  ✅ Include
├── markdown_renderer.py          ✅ Include
├── requirements.txt             ✅ Include (dependencies)
├── README.md                    ✅ Include (documentation)
├── LICENSE                      ✅ Include (MIT license)
├── QUICKSTART.md                ✅ Include (setup guide)
├── .gitignore                   ✅ Include (tells Git what to skip)
│
├── assets/                      ✅ Include (if you have custom assets)
├── main.spec                    ❌ Skip (auto-generated by PyInstaller)
├── build/                       ❌ Skip (auto-generated build files)
├── dist/                        ❌ Skip (contains .exe — too large)
├── __pycache__/                 ❌ Skip (Python cache)
└── data/                        ❌ Skip (user data, session files)
```

The **.gitignore** file we created automatically skips the ❌ items, so you don't need to worry about them.

2. You should see:
   - All your Python files
   - README.md displayed at the bottom (the overview)
   - File list on the left

## Step 6 (Optional): Add a Release

If you want people to download your .exe:

1. Go to your GitHub repo
2. Click "Releases" → "Create a new release"
3. **Tag version**: `v1.0` (or whatever version)
4. **Release title**: `EleViewer v1.0`
5. **Description**: Paste the feature list from README.md
6. **Upload binary**: Click "Attach binaries" and upload your `main.exe`
7. Click "Publish release"

Now people can download your .exe directly from GitHub!

## Troubleshooting

### "git is not recognized"

Git is not installed. Download it from: https://git-scm.com/download/win

### "Authentication failed"

Try using a Personal Access Token instead of password:

1. Go to GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Click "Generate new token"
3. Give it permissions: `repo`, `workflow`
4. Copy the token
5. When Git asks for password, paste the token instead

### "fatal: remote origin already exists"

If you get this error, run:
```bash
git remote remove origin
git remote add origin https://github.com/YOUR_USERNAME/eleviewer.git
```

### Files not showing on GitHub after push

1. Check `.gitignore` — it might be excluding them
2. Verify files were committed: `git log` (shows recent commits)
3. Try: `git push origin main`
